## 1. 현황 분석

### 1.1 현재 상태

| 항목 | 현재 | 문제점 |
|------|------|--------|
| 소스 저장소 | Google Drive (로컬 동기화) | 버전 관리 불가, 동시 편집 충돌 위험 |

### 1.2 기술검토서 원안

- **소스 관리**: GitHub Private Repository
- **호스팅**: Cloudflare Pages
- **CI/CD**: GitHub Actions → Cloudflare Pages
- **도메인**: mumuki.net (Cloudflare DNS)

## 2. 소스코드 저장소 전략

| 항목 | GitHub Private | GitHub Public | GitLab | 현행 (Google Drive) |
|------|---------------|--------------|--------|-------------------|
| 비용 | 무료 | 무료 | 무료 | 무료 |
| 비공개 | **O** | X | **O** | **O** |
| CI/CD | GitHub Actions | GitHub Actions | GitLab CI | X |
| Cloudflare 연동 | **네이티브** | **네이티브** | 수동 | X |
| 협업 | PR/Issue/Review | PR/Issue/Review | MR | 충돌 위험 |

| 민감 데이터 보호 | .gitignore | **불가** (공개) | .gitignore | 물리적 분리 |

### 2.2 추천: GitHub Private Repository

**이유:**

1. **민감 데이터 보호**: 설립안, 향후 회계 데이터가 포함된 프로젝트에 Private 필수
2. **Cloudflare Pages 네이티브 연동**: 1클릭으로 자동 배포 파이프라인 구축
3. **비영리 단체 무료**: GitHub for Nonprofits 프로그램 (Team 플랜 무료 가능)
4. **Quarto 생태계**: Quarto Pub, GitHub Actions 등 공식 워크플로우가 GitHub 기준

## 3. 배포 전략

### 3.1 옵션 비교

| 항목 | Cloudflare Pages | GitHub Pages | Netlify | Vercel |
|------|-----------------|-------------|---------|--------|
| 비용 | **무료** (무제한) | 무료 (1GB) | 무료 (100GB) | 무료 (100GB) |
| 대역폭 | **무제한** | 100GB/월 | 100GB/월 | 100GB/월 |
| 한국 CDN | **O** (300+ PoP) | O | O | O |
| 커스텀 도메인 | **O** | O | O | O |
| HTTPS | **자동** | 자동 | 자동 | 자동 |
| 서버 함수 | **Workers** (무료 10만/일) | X | Functions | Functions |
| 빌드 | **서버** (500/월) | GitHub Actions | 서버 | 서버 |
| 프리뷰 배포 | **브랜치별 자동** | X | 브랜치별 | 브랜치별 |

### 3.2 추천: Cloudflare Pages (기술검토서 원안 유지)

**결정적 이유:**

1. **대역폭 무제한**: 기부 캠페인 등 트래픽 급증 시에도 비용 0원
2. **Workers 통합**: 2~3차년 토스페이먼츠 결제 연동 시 같은 플랫폼에서 처리
3. **한국 CDN**: 주 이용자가 한국인이므로 응답속도 중요
4. **프리뷰 배포**: PR 생성 시 자동으로 별도 URL 생성 → 검토 용이
5. **연간 비용**: ₩0 (도메인 비용만 별도)

### 3.3 배포 아키텍처


## 4. 단계별 실행 계획

### Step 1: GitHub 저장소 생성 (10분)

```bash
# 1. GitHub에서 Private 리포지토리 생성
#    이름: mumuki.net (또는 mumuki-website)
#    Visibility: Private
#    .gitignore: 없음 (이미 로컬에 있음)

# 2. 로컬에서 리모트 연결
cd /path/to/mumuki
git remote add origin git@github.com:<org>/mumuki.net.git

# 3. .gitignore 업데이트 (docs/ 추가)
# 4. 변경사항 커밋 & 푸시
git add -A
git commit -m "[인프라] GitHub 저장소 연결 및 프로젝트 구조 정리"
git push -u origin main
```

### Step 2: Cloudflare Pages 연결 (15분)

```
1. Cloudflare Dashboard → Pages → Create a project
2. "Connect to Git" → GitHub 계정 연결 → mumuki.net 리포 선택
3. Build settings:
   - Framework preset: None
   - Build command: quarto render
   - Build output directory: docs
   - Root directory: / (기본값)
4. Environment variables:
   - QUARTO_VERSION: 1.9.16
5. "Save and Deploy" 클릭
```

> **참고**: Cloudflare Pages는 Quarto를 네이티브로 지원하지 않으므로,  
> GitHub Actions에서 빌드 후 결과를 배포하는 방식이 더 안정적이다 (Step 3 참조).

### Step 3: GitHub Actions CI/CD (추천 방식)

`.github/workflows/deploy.yml`:

```yaml
name: Deploy to Cloudflare Pages

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      deployments: write

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Quarto
        uses: quarto-dev/quarto-actions/setup@v2
        with:
          version: "1.9.16"

      - name: Render
        run: quarto render

      - name: Deploy to Cloudflare Pages
        if: github.ref == 'refs/heads/main'
        uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          command: pages deploy docs --project-name=mumuki-net
```

**필요한 GitHub Secrets:**

| Secret 이름 | 설명 | 발급 방법 |
|-------------|------|-----------|
| `CLOUDFLARE_API_TOKEN` | Pages 배포 권한 토큰 | Cloudflare Dashboard → API Tokens → Create Token |
| `CLOUDFLARE_ACCOUNT_ID` | 계정 ID | Cloudflare Dashboard → 우측 사이드바 |

### Step 4: 도메인 연결 (5분)

```
1. Cloudflare Dashboard → Pages → mumuki-net → Custom domains
2. "Add custom domain" → mumuki.net 입력
3. DNS가 이미 Cloudflare에 있으면 자동 설정
4. DNS가 외부에 있으면 네임서버를 Cloudflare로 변경:
   - NS: xxx.ns.cloudflare.com
   - NS: yyy.ns.cloudflare.com
5. HTTPS: Universal SSL 자동 적용 (10분 이내)
6. www.mumuki.net → mumuki.net 리다이렉트 Rule 추가
```

## 5. 대안: GitHub Pages (간소화 방식)

Cloudflare Pages 설정이 부담스러울 경우, GitHub Pages로 먼저 시작할 수 있다.

### 장점

- 설정이 매우 간단 (repo Settings → Pages → Deploy from branch)
- `docs/` 폴더를 빌드 출력으로 그대로 사용 가능
- GitHub Actions로 자동 빌드 가능

### 단점

- **서버 함수 없음**: 2~3차년 결제 연동 시 별도 서비스 필요
- **대역폭 100GB/월 제한**: 대부분의 비영리 사이트에는 충분
- **Private 리포에서 GitHub Pages**: GitHub Pro 또는 GitHub Free (2024.1월부터 지원)

### 실행 방법

```yaml
# .github/workflows/deploy-ghpages.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: quarto-dev/quarto-actions/setup@v2
      - run: quarto render

      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

```
GitHub Settings → Pages → Source: GitHub Actions
Custom domain: mumuki.net (CNAME 레코드 설정)
```

## 9. 의사결정 요약

| 결정 사항 | 선택 | 대안 | 전환 용이성 |
|-----------|------|------|-------------|
| 소스 저장소 | **GitHub Private** | GitLab | 높음 (git remote 변경) |
| 배포 플랫폼 | **Cloudflare Pages** | GitHub Pages | 높음 (CI/CD 변경) |
| CI/CD | **GitHub Actions** | Cloudflare 직접 빌드 | 높음 |

> 세 가지를 분리하여 보안과 편의성을 모두 확보한다.
