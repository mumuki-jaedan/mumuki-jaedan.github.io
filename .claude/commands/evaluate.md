mumuki.net 웹사이트의 결과물을 설립안 부합성과 기술 적합성 관점에서 평가합니다.

## 평가 축

### 1. 설립안 부합성 평가

설립안(2026.02.28)의 핵심 요구사항 대비 구현 현황을 평가한다:

| 요구사항 | 설립안 근거 | 평가 기준 |
|----------|-------------|-----------|
| 다이렉트 기부 구조 설명 | §1 | 구조도/흐름도가 설립안 내용을 정확히 반영하는가 |
| 연차별 사업 계획 반영 | §2 | 1/2/3차년 로드맵이 사이트에 올바르게 표현되는가 |
| 회계 전부 공개 | §3 | 계좌/카드 내역 공개 기능이 구현되었는가 |
| 비식별화 처리 | §3 | 기부자/수혜자명이 적절히 마스킹되는가 |
| 매칭 지출 내역 | §3 | 가상계좌 입금 ↔ 지출 매칭이 병기되는가 |
| 비용 분리 | §3 | 사업수행비 vs 운영비 구분이 명확한가 |
| 조직 구성 | §3 | 이사회/사무국/회장단/감사 구조가 표현되는가 |
| 도서관 동행 | §4.1 | 프로그램 설명이 설립안과 일치하는가 |
| 직업 교육 | §4.2 | 프로그램 설명이 설립안과 일치하는가 |

### 2. 기술 적합성 평가

기술검토서(tech_document/)의 아키텍처 결정 사항 대비 구현 현황:

| 항목 | 평가 기준 |
|------|-----------|
| Quarto 설정 | _quarto.yml이 기술검토서의 설계를 따르는가 |
| 디자인 시스템 | 색상/타이포/간격이 design-system.md를 준수하는가 |
| 프로젝트 구조 | 폴더/파일 구조가 기술검토서 §2.3을 따르는가 |
| 반응형 | 모바일/태블릿/데스크톱 대응이 되는가 |
| 성능 | Lighthouse 점수 90+ 달성 가능한가 |
| 보안 | 비식별화, 데이터 분리(.gitignore)가 적절한가 |

### 3. 사용자 경험 평가

| 항목 | 평가 기준 |
|------|-----------|
| 네비게이션 | 3클릭 이내에 모든 페이지 접근 가능한가 |
| 가독성 | 본문 65ch 이내, 줄간격 1.7, 적절한 여백 |
| CTA | 후원하기 버튼이 항상 접근 가능한가 |
| 신뢰감 | 투명성 정보가 쉽게 찾을 수 있는가 |

## 수행 절차

1. PROGRESS.md에서 전체 진행 현황을 파악한다.
2. 설립안 원문과 구현 결과를 항목별로 대조한다.
3. 기술검토서와 실제 구현을 대조한다.
4. 각 항목에 점수(1-5)를 부여하고 종합 평가한다.
5. PLAN.md에 개선 필요 사항을 반영한다.

## 출력 형식

```
## 평가 결과

### 설립안 부합성: X/5
[항목별 점수 테이블]

### 기술 적합성: X/5
[항목별 점수 테이블]

### UX 평가: X/5
[항목별 점수 테이블]

### 종합 점수: X/5

### 개선 권고사항

2. ...

### 다음 Phase 준비도
[다음 단계로 넘어갈 수 있는지 판단]
```

사용자의 입력: $ARGUMENTS
