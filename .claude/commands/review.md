mumuki.net 웹사이트의 결과물을 품질 관점에서 검토합니다.

## 검토 항목

### 1. 코드 품질
- Quarto YAML 문법 오류 여부
- SVG 유효성 (viewBox, 닫히지 않은 태그)
- SCSS 컴파일 가능 여부
- 파일/폴더 네이밍 일관성

### 2. 디자인 시스템 준수
- 브랜드 색상 코드 정확성 (#2C5F5C, #E07A5F, #D4A574)
- Tufte 스타일 원칙 (최소주의, 데이터-잉크 비율)
- 타이포그래피 (Noto Serif KR, Noto Sans KR)
- 간격 체계 (8px 그리드)

### 3. 콘텐츠 정확성
- 설립안 내용과의 일치 여부
- 오탈자 및 문법 오류
- 링크 유효성
- 이미지 대체 텍스트 (alt) 존재 여부

### 4. 기술 적합성
- 반응형 디자인 대응
- 접근성 (WCAG 2.1 AA)
- SEO 기본 요소 (meta, title, description)
- 성능 (이미지 최적화, 불필요한 리소스)

## 수행 절차

1. PROGRESS.md에서 최근 완료된 작업을 확인한다.
2. 해당 파일들을 읽고 위 4개 항목별로 검토한다.
3. 발견된 이슈를 심각도(높음/중간/낮음)로 분류한다.
4. 수정 필요 사항을 구체적으로 제시한다.

## 출력 형식

```
## 리뷰 결과

### 검토 범위
[검토한 파일 목록]

### 이슈 목록

|---|--------|------|------|-----------|
| 1 | 높음   | ... | ... | ... |

### 통과 항목
- [문제 없는 항목들]

### 종합 의견
[전체적인 품질 평가와 개선 방향]
```

사용자의 입력: $ARGUMENTS
